PRACTICA 3: Introducción a BISON/YACC.

Mini-tutorial de BISON


Se presentará el proceso de generación de analizadores sintácticos en C a partir de una especificación BISON.

1. Compilación de una especificación BISON.

Compilar las dos especificaciones BISON siguientes y comprobar su funcionamiento.


EJEMPLO 1

Minianalizador sintáctico para frases en español. 

%token ARTICULO, NOMBRE, ADJETIVO, PREPOSICION, VERBO 

%% 

dialogo : frase 
        | dialogo  frase  
; 

frase : '\n' 
      | sujeto predicado  '\n'  {printf("\n>> Frase correcta\n");} 
; 

sujeto : frase_nominal 
; 

frase_nominal : NOMBRE 
              | ARTICULO NOMBRE 
              | ARTICULO NOMBRE ADJETIVO 
              | ARTICULO NOMBRE frase_preposicional 
;                

frase_preposicional : PREPOSICION frase_nominal 
; 

predicado : frase_verbal 
; 

frase_verbal : VERBO  
             | VERBO frase_nominal 
             | VERBO frase_preposicional 
;           
              
%% 
               
int main() 
{ 
    yyparse(); 
} 
              
void yyerror (char *s) 
{ 
    printf ("%s\n", s); 
               
} 

int yywrap()   
{   
    return 1;   
}

Descargad y descomprimid el fichero ejemplo1.tgz.


COMPILACIÓN 

   $ bison -d ejemplo1.y 
   $ flex ejemplo1.l  
   $ gcc -o ejemplo1 ejemplo1.tab.c lex.yy.c

Tambien es posible utilizar el comando make (se incluye fichero Makefile).


EJEMPLO 2

Calculadora básica para números enteros. 

%{ 
#define YYSTYPE int
#include <math.h>    
%} 

/* Declaraciones de BISON */ 
%token ENTERO 
%left '-' '+' 
%left '*' '/' 

%%
               
/* Gramática */ 

input:     /* cadena vacía */ 
          | input line 
; 
               
line:      '\n' 
          | exp '\n'  { printf ("\t%d\n", $1); } 
; 
               
exp:       ENTERO             { $$ = $1;         } 
          | exp '+' exp        { $$ = $1 + $3;    } 
          | exp '-' exp        { $$ = $1 - $3;    } 
          | exp '*' exp        { $$ = $1 * $3;    } 
          | exp '/' exp        { $$ = $1 / $3;    } 
; 

%% 

int main() 
{ 
    yyparse(); 
} 
              
void yyerror (char *s) 
{ 
    printf ("%s\n", s); 
               
} 

int yywrap()   
{   
    return 1;   
}

Descargad y descomprimid el fichero ejemplo2.tgz.


COMPILACIÓN 

   $ bison -d ejemplo2.y 
   $ flex ejemplo2.l  
   $ gcc -o ejemplo2 ejemplo2.tab.c lex.yy.c -lm

Tambien es posible utilizar el comando make (se incluye fichero Makefile).

2. Ejercicio práctico.

Escribir las especificaciones bison y flex, además del código adicional que sea necesario, para extender la calculadora del último ejemplo de acuerdo a las siguientes especificaciones :
  • Modificar el analizador léxico para que acepte y maneje números reales.
  • Incluir los operadores "-" (cambio de signo) y "^" (exponenciación) y soportar el uso de paréntesis para agrupar expresiones y cambiar su asociatividad.
  • Incluir la posibilidad de definir variables y de utilizarlas en los cálculos.
    • Los nombres de variable serán secuencias de caracteres alfanuméricos más el "_", con la restricción de que no podrán comenzar por un número.
    • No se distinguen mayúsculas y minúscula. (Ej.: Velocidad, velocidad y VeLoCiDaD son el mismo identificador)
    • La declaración de variables será de la forma : <identificador_variable> = <valor> (donde valor puede ser una constante numérica o el resultado de una expresion aritmética)
    • Será necesario implementar algún tipo de estructura que haga la función de "tabla de símbolos" y que relacionará cada nombre de variable con el valor que tenga asociado en ese momento. La elección de las estructras de almacenamiento es libre.(1)
    • Será posible redefinir el valor de las variables una vez creadas, sobreescribiendo el valor anterior.
    • Si el usuario indica un nombre de variable que no existe se le informará con un mensaje de error indicando que la variable no está definida y se continuará la ejecución.


    • NOTAS :

    (1) En el manual de BISON se incluye un ejemplo similar a este, aunque algo más complejo, que utiliza una lista enlazada para implementar la tabla de símbolos.

    La forma más sencilla de implementar la tabla es usar un array de pares (nombre, valor) y fijar un límite en el número de variables permitidas (por ejemplo, 30) y en caso de sobrepasarlo informar del error.

    (2) Aunque la calculadora debe admitir tanto números reales como enteros, se recomienda que, para evitar problemas de conversión de tipos, la aritmetica interna del programa se realize con números reales.
